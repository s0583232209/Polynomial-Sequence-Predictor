# PolySeqNet — Polynomial Sequence Predictor

**A PyTorch LSTM that learns to predict the continuation of polynomial integer sequences — without ever being told the formula.**

```
Input:  5, 8, 13, 20, 29, 40, 53, 68   (8 terms)
Output: 85, 104, 125                   (next 3 terms)
```

The sequence above is generated by `n² + 2n + 7`, but the model is never given that formula. It only ever sees numbers. It has to infer, purely from patterns in the training data, how a polynomial sequence behaves — and extrapolate it correctly.

This is a Computer Science / AI university project built end-to-end: synthetic data generation, a custom PyTorch `Dataset`, an LSTM regression model, a training loop with early stopping, a CLI predictor, and a desktop GUI.

## How it works

1. **`data/dataset.py`** — a synthetic dataset of polynomial sequences (degrees 1–5) is read from CSV. Each row is turned into an 8-value input tensor and a 3-value target tensor, then **per-sample normalized** (subtract the mean, divide by the std of that specific input sequence). This matters a lot here: polynomial sequences can range from single digits to enormous numbers depending on degree and starting index, so normalizing per-sequence (rather than globally) keeps the LSTM's inputs on a comparable scale regardless of how big the underlying numbers are.
2. **`models/lstm.py`** — a 2-layer LSTM consumes the 8 (normalized) input values one at a time and predicts the next 3 in one shot from its final hidden state.
3. **`train.py`** — trains the model with `AdamW`, gradient clipping, an LR scheduler, and early stopping, saving the best checkpoint to `best_model.pth`.
4. **`predict.py`** — loads `best_model.pth` and exposes a single function, `predict_sequence(sequence)`, that normalizes an 8-number input the same way, runs inference, and de-normalizes the output back to the original scale. It also works as a standalone CLI script.
5. **`main.py` + `gui/`** — a PyQt5 desktop app around `predict.py`: type in 8 numbers (or load the example sequence), hit Predict, and see the 3 predicted values plus a small chart of the input sequence and its predicted continuation.

## The dataset

Each row of the dataset describes one polynomial sequence sample:

| Column | Meaning |
|---|---|
| `degree` | Polynomial degree (1–5) |
| `formula` | Human-readable formula, e.g. `n²+2n+7` |
| `coefficients` | The polynomial's coefficients |
| `start_n` | Starting index `n` the sequence begins at |
| `input_length` | Always 8 |
| `target_length` | Always 3 |
| `input_sequence` | The 8 known terms, comma-separated |
| `target_sequence` | The 3 terms to predict, comma-separated |

**Example row:**

| degree | formula | input_sequence | target_sequence |
|---|---|---|---|
| 2 | n²+2n+7 | 4,7,12,19,28,39,52,67 | 84,103,124 |

Design intent behind the dataset (10,000 samples per degree, 50,000 total; degrees 1–5; random starting index in `[-20, 30]`; coefficients that can be positive, negative, or zero except the leading one; no duplicate sequences):

- **Balanced across degrees** so the model isn't biased toward easy (low-degree) sequences.
- **Random `start_n`** so the model can't just memorize "sequence #n always looks like this" — it has to actually read the shape of the numbers it's given.
- **No duplicates**, generated by tracking seen sequences in a Python `set()` inside a `while True` loop (rather than recursion), so the same exact 8-number input never appears twice.

`data/tools/database_explore.py` reports sample count, missing values, duplicate rows, and the degree distribution — a quick sanity check after generating or receiving a dataset. `data/tools/database_split.py` then does a **stratified** 80/10/10 split by `degree`, so every split has the same proportion of each polynomial degree.

## The model

`models/lstm.py` — `PolynomialLSTM`:

```
Input (8 values, 1 feature each)
        ↓
   LSTM (2 layers, hidden_size=64, dropout=0.2)
        ↓
   take final hidden state
        ↓
   Linear(64 → 32) → ReLU → Linear(32 → 3)
        ↓
Output (3 predicted values)
```

The LSTM reads the 8 input numbers step by step and the final hidden state is passed through a small feed-forward head to produce all 3 future values at once (rather than one-at-a-time autoregressive decoding).

## Training

`train.py`:

- Loads data via `utils/dataloaders.py` → `create_dataloaders()` (batch size 32).
- Optimizer: `AdamW` (`lr=0.001`, `weight_decay=1e-4`).
- `ReduceLROnPlateau` scheduler (halves the LR after 5 epochs without validation improvement).
- Gradient clipping (`max_norm=1.0`) to avoid exploding gradients.
- Up to 200 epochs, with **early stopping** after 25 epochs without validation improvement.
- Saves the best-performing checkpoint (by validation loss) to `best_model.pth`.
- After training, reloads the best checkpoint, evaluates it on the held-out test set, prints test MSE/MAE, and plots train vs. validation loss.

Run it with:

```bash
python train.py
```

## Predicting from the command line

`predict.py` can be run directly:

```bash
python predict.py
```

```
Using device: cpu
Model loaded successfully!
Enter 8 numbers separated by commas:
5, 8, 13, 20, 29, 40, 53, 68
Predicted next values:
85.02
104.13
124.87
```

Internally, `predict_sequence(sequence: list[float]) -> list[float]` normalizes the input using its own mean/std, runs it through the loaded model, and rescales the output back to the original numbers — the same normalization strategy used during training, so the model always sees inputs on a comparable scale no matter how large the actual sequence values are.

## Desktop GUI

`main.py` launches a PyQt5 application (`gui/main_window.py`) with:

- 8 labeled input boxes (`a₀`…`a₇`) for the known sequence, plus a **"Load example sequence"** button.
- A **Predict** button that runs inference on a background thread (`gui/predict_worker.py`) so the window never freezes.
- 3 output boxes (`a₈`, `a₉`, `a₁₀`) showing the predicted continuation.
- An embedded matplotlib chart (`gui/graph_canvas.py`) plotting the input sequence in blue and the predicted continuation in coral, so you can see the curve visually.
- A status bar that reports whether `best_model.pth` was found, and surfaces prediction errors as dialogs.

Run it with:

```bash
python main.py
```

> `predict.py` is imported before PyQt5 in `main.py` — on some Windows setups, importing PyQt5 first changes the DLL search path in a way that breaks `torch`'s own DLL loading, so the import order matters.
